# Kosmos 部署指南 v0.2.0

本指南将引导您完成 Kosmos 后端服务的部署，包括所有必需的依赖服务和环境配置。

## 1. 核心依赖

成功部署和运行 Kosmos 后端需要以下两个核心外部依赖：

1.  **PostgreSQL 数据库**: 用于存储所有元数据，包括用户信息、知识库配置、文档信息、文本片段和索引标签等。
2.  **Milvus 向量数据库**: 用于存储文本片段的向量嵌入（Embeddings），是实现语义搜索的关键。

## 2. 环境准备

在开始之前，请确保您的系统上安装了以下软件：

-   **Python**: 3.9 或更高版本。
-   **Docker** 和 **Docker Compose**: (推荐) 用于快速启动 PostgreSQL 和 Milvus。
-   **Git**: 用于克隆项目仓库。

## 3. 部署依赖服务 (推荐使用 Docker)

我们强烈建议使用 Docker Compose 来一键启动 PostgreSQL 和 Milvus 服务，这能极大地简化环境搭建的复杂性。

1.  **创建 `docker-compose.yml` 文件**:
    在您的项目根目录下创建一个名为 `docker-compose.yml` 的文件，并填入以下内容。此配置将同时启动 PostgreSQL 和 Milvus。

    ```yaml
    version: '3.8'

    services:
      postgres:
        image: postgres:15
        container_name: kosmos_postgres
        environment:
          POSTGRES_USER: kosmos_user
          POSTGRES_PASSWORD: kosmos_password
          POSTGRES_DB: kosmos_db
        ports:
          - "5432:5432"
        volumes:
          - ./volumes/postgres:/var/lib/postgresql/data
        restart: always

      etcd:
        container_name: kosmos_milvus_etcd
        image: quay.io/coreos/etcd:v3.5.5
        environment:
          - ETCD_AUTO_COMPACTION_MODE=revision
          - ETCD_AUTO_COMPACTION_RETENTION=1000
          - ETCD_QUOTA_BACKEND_BYTES=4294967296
        volumes:
          - ./volumes/etcd:/etcd
        command: etcd -advertise-client-urls=http://127.0.0.1:2379 -listen-client-urls http://0.0.0.0:2379 --data-dir /etcd
        restart: always

      minio:
        container_name: kosmos_milvus_minio
        image: quay.io/minio/minio:RELEASE.2023-03-20T20-16-18Z
        environment:
          MINIO_ROOT_USER: root
          MINIO_ROOT_PASSWORD: minio_password
        volumes:
          - ./volumes/minio:/data
        command: minio server /data
        healthcheck:
          test: ["CMD", "curl", "-f", "http://localhost:9000/minio/health/live"]
          interval: 30s
          timeout: 20s
          retries: 3
        restart: always

      milvus:
        container_name: kosmos_milvus
        image: milvusdb/milvus:v2.3.10
        depends_on:
          - etcd
          - minio
        ports:
          - "19530:19530"
          - "9091:9091"
        environment:
          ETCD_ENDPOINTS: etcd:2379
          MINIO_ADDRESS: minio:9000
        volumes:
          - ./volumes/milvus:/var/lib/milvus
        restart: always

    volumes:
      postgres:
      etcd:
      minio:
      milvus:
    ```

2.  **启动服务**:
    在包含 `docker-compose.yml` 文件的目录下，执行以下命令：
    ```bash
    docker-compose up -d
    ```
    等待几分钟，直到所有服务都成功启动。您可以使用 `docker-compose ps` 命令查看服务状态。

## 4. 后端应用部署

1.  **克隆项目**:
    ```bash
    git clone <your-repository-url>
    cd Kosmos
    ```

2.  **创建并激活 Python 虚拟环境**:
    ```bash
    python3 -m venv .venv
    source .venv/bin/activate
    # 对于 Windows 用户: .venv\Scripts\activate
    ```

3.  **安装依赖**:
    ```bash
    pip install -r requirements.txt
    ```

4.  **配置环境变量**:
    后端服务的配置通过根目录下的 `.env` 文件加载。请复制 `.env.example` 文件为 `.env`，并根据您的环境修改其中的值。

    ```bash
    cp .env.example .env
    ```

    **请务必检查并修改 `.env` 文件中的以下变量**:

    -   **安全密钥 (重要!)**:
        `JWT_SECRET_KEY`, `JWT_REFRESH_SECRET_KEY`, `CREDENTIAL_ENCRYPTION_KEY` 必须被替换为**高强度随机字符串**。您可以使用以下命令生成：
        ```bash
        # 在 Linux/macOS 或 Git Bash 中执行
        openssl rand -hex 32
        ```
        请为这三个变量分别生成不同的值。

    -   **数据库配置**:
        如果您使用了我们提供的 `docker-compose.yml`，请按如下配置：
        ```dotenv
        POSTGRES_USER="kosmos_user"
        POSTGRES_PASSWORD="kosmos_password"
        POSTGRES_HOST="localhost"
        POSTGRES_PORT="5432"
        POSTGRES_DB="kosmos_db"
        ```
        如果您使用自己的 PostgreSQL 实例，请填写对应的连接信息。

    -   **Milvus 配置**:
        如果您使用了我们提供的 `docker-compose.yml`，请按如下配置：
        ```dotenv
        MILVUS_HOST="localhost"
        MILVUS_PORT="19530"
        MILVUS_TOKEN="root:minio_password" # Milvus 2.3+ 默认使用 root:minio_password
        ```
        如果您使用自己的 Milvus 实例，请填写对应的连接信息。

    -   **Embedding 模型维度**:
        ```dotenv
        OPENAI_EMBEDDING_DIM="1024"
        ```
        请确保此维度与您计划使用的 Embedding 模型输出维度一致。例如，OpenAI `text-embedding-ada-002` 是 1536，`text-embedding-3-small` 是 1024。**此值在创建第一个知识库后不能轻易更改**，因为它决定了 Milvus Collection 的 Schema。

    -   **去重配置**:
        `DEDUP_*` 系列变量控制文档上传时的去重逻辑，可根据需要调整。

5.  **启动应用**:

    -   **开发环境**:
        使用 `uvicorn` 启动，支持热重载。
        ```bash
        uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
        ```

    -   **生产环境**:
        推荐使用 `gunicorn` 配合 `uvicorn` 的 worker 类来运行。
        ```bash
        gunicorn app.main:app --workers 4 --worker-class uvicorn.workers.UvicornWorker --bind 0.0.0.0:8000
        ```

## 5. 验证部署

应用启动后，数据库表会自动创建。您可以访问自动生成的 API 文档页面来验证后端服务是否正常运行：

[http://localhost:8000/docs](http://localhost:8000/docs)

如果能看到 API 文档页面，说明后端服务已成功部署。
