
# Kosmos 知识库系统 - README

欢迎来到 Kosmos，一个为现代企业和团队设计的、可插拔的语义知识库系统。Kosmos 旨在帮助您构建、管理和利用大规模多模态记忆，将非结构化数据转化为可操作的智能。

## 核心价值

- **语义化**: 超越关键词搜索，Kosmos 理解您数据的深层含义，提供更精准、更智能的搜索结果。
- **多模态**: 无缝处理和索引各类数据，包括文本文档、图片、代码片段等，构建统一的知识视图。
- **可插拔架构**: 灵活的模块化设计，允许您轻松集成不同的模型服务（如 Embedding, Reranking, LLM），以适应不断变化的技术需求。
- **企业级**: 提供完善的用户权限管理、多租户知识库和异步任务处理机制，满足企业级应用的稳定性和安全性要求。

## 核心概念

Kosmos 的知识体系构建在四个核心抽象之上，它们层层递进，构成了从原始数据到可检索知识的完整链条。

1.  **知识库 (Knowledge Base)**
    - **定义**: 知识库是最高层级的容器，代表一个独立的知识领域或项目空间。每个知识库拥有自己的成员、权限设置、模型配置和标签体系。
    - **特点**:
        - **多租户**: 不同知识库之间的数据和配置是隔离的。
        - **可配置性**: 每个知识库可以独立配置其使用的 Embedding、Reranker、LLM 等 AI 模型。
        - **标签体系 (Tag Dictionary)**: 知识库可以定义一个层级化的标签字典，用于后续对内容的精准分类和筛选。

2.  **文档 (Document)**
    - **定义**: 文档是用户上传到知识库的原始资料，是知识的直接来源。Kosmos 将文档分为 **逻辑文档** 和 **物理文档**。
    - **逻辑文档 (Document)**: 代表用户上传行为的记录，包含文件名、类型等元数据。
    - **物理文档 (PhysicalDocument)**: 代表实际存储的文件。系统通过内容哈希（SHA256）实现文件去重，多个逻辑文档可以指向同一个物理文档，有效节省存储空间。

3.  **片段 (Fragment)**
    - **定义**: 片段是文档经过智能解析后产生的最小知识单元。它是系统进行理解、索引和检索的基础。
    - **类型**:
        - **文本 (Text)**: 从文档中提取的段落、句子或代码块。
        - **截图 (Screenshot)**: 从文档中捕获的页面截图。
        - **图表 (Figure)**: 从文档中识别出的独立图、表、流程图等。
    - **特点**: 每个片段都保留了其在原始文档中的位置信息（如页码），为上下文追溯提供了可能。

4.  **索引 (Index)**
    - **定义**: 索引是对 **文本片段 (Text Fragment)** 进行深度加工后产生的可检索记录。它是实现语义搜索的核心。
    - **构成**:
        - **向量嵌入 (Embedding)**: 片段内容的数学表示，用于计算语义相似度。
        - **标签 (Tags)**: 基于知识库的标签体系和 LLM 理解能力，为片段自动生成的关键词或分类。
        - **元数据**: 其他用于检索和过滤的信息。
    - **存储**: 索引数据存储在关系型数据库（PostgreSQL）中，而向量嵌入则存储在专门的向量数据库（如 Milvus）中，以实现高效的相似度检索。

## 系统架构

Kosmos 采用经典的三层分离架构，确保了代码的高内聚、低耦合，易于维护和扩展。

- **Router (路由层)**: 位于 `app/routers`。负责接收外部 HTTP 请求，验证请求参数，并调用相应的服务层处理。它定义了系统的 API 接口。
- **Service (服务层)**: 位于 `app/services`。包含系统的核心业务逻辑。服务层编排不同的数据模型和外部服务（如 AI 模型），完成具体的业务功能，如用户认证、文档上传、内容解析、语义搜索等。
- **Model (模型层)**: 位于 `app/models`。定义了系统的数据结构和数据库表映射（使用 SQLAlchemy ORM）。它构成了系统的骨架。

**数据与任务流**:

1.  **异步任务处理**: 对于耗时操作，如文档解析和批量索引，Kosmos 采用基于 `asyncio` 的异步任务队列 (`UnifiedJobService`)。请求被立即响应，任务在后台执行，用户可以通过 `jobs` API 查询任务状态。
2.  **AI 模型集成**: 系统通过 `CredentialService` 和 `KBModelConfig` 管理和配置对外部 AI 模型的访问。这使得更换或升级模型变得简单，只需修改配置，无需改动核心代码。

## 主要功能

- **用户与认证管理** (`auth`, `users`):
    - 用户注册、登录、登出。
    - JWT (JSON Web Token) 认证与刷新机制。
    - 基于角色的权限控制（普通用户、管理员、系统管理员）。

- **知识库管理** (`knowledge_bases`):
    - 创建、更新、删除知识库。
    - 管理知识库成员及其角色（所有者、管理员、成员）。
    - 配置知识库的 AI 模型和标签字典。

- **凭证管理** (`credentials`):
    - 安全地存储和管理访问各类 AI 模型所需的 API Key 等凭证（通过加密存储）。
    - 支持多种模型类型 (Embedding, Reranker, LLM, VLM)。

- **文档管理** (`documents`):
    - 上传、下载、删除文档。
    - 批量管理文档。
    - 查看文档的解析和索引状态。

- **内容解析** (`parser`):
    - 异步解析上传的文档，自动生成文本、截图、图表等多种类型的片段。
    - 支持强制重新解析。
    - 提供文档解析状态的查询。

- **索引管理** (`index`):
    - 异步地为文本片段创建向量索引和标签。
    - 支持对单个片段、单个文档或批量文档发起索引任务。
    - 提供知识库的索引统计信息。

- **语义搜索** (`search`):
    - **混合搜索**: 结合向量相似度和标签过滤，实现精准的语义搜索。
    - **高级查询语法**: 支持在搜索框中直接使用 `+tag` (必须包含), `-tag` (必须不包含), `~tag` (偏好) 等语法进行高级筛选。
    - **上下文感知**: 搜索结果可以关联返回相关的截图和图表，提供更丰富的上下文。

- **任务管理** (`jobs`):
    - 提供对所有后台异步任务（如解析、索引）的统一查询和管理接口。
    - 查看任务详情、进度和执行结果。

## 快速开始

1.  **启动依赖服务 (推荐)**
    本项目依赖 PostgreSQL 和 Milvus。我们提供了一个 `docker-compose.yml` 文件来一键启动这两个服务。
    ```bash
    # 这将会在后台启动 PostgreSQL 和 Milvus 容器
    docker-compose up -d
    ```

2.  **配置环境变量**
    复制环境变量模板文件，并根据需要（特别是安全密钥）进行修改。
    ```bash
    cp .env.example .env
    ```
    **重要**: 请务必在 `.env` 文件中为 `JWT_SECRET_KEY`, `JWT_REFRESH_SECRET_KEY`, 和 `CREDENTIAL_ENCRYPTION_KEY` 生成新的高强度随机值。

3.  **安装 Python 依赖**
    ```bash
    pip install -r requirements.txt
    ```

4.  **启动后端应用**
    ```bash
    uvicorn app.main:app --reload
    ```
    服务启动后，您可以通过 `http://127.0.0.1:8000/docs` 访问 API 文档。

有关更详细的部署选项和环境变量说明，请参阅 [部署指南.md](./部署指南.md)。
